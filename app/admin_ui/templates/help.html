Glossary

Plain-language definitions of the terms used across this console and the setup wizard.

Backend
The search engine that actually stores your records — usually Elasticsearch or OpenSearch. EGG-API does not host your data; it reads from the backend.
Index
The pool of records inside the backend. Think of it as a collection or a table name. One EGG-API instance exposes one index.
Record
A single entry — one notice, one object, one photograph — as it appears in your catalogue.
Mapping
The translation between backend field names (e.g. creator_csv) and the public field names EGG exposes (e.g. creators).
Facet
A filterable breakdown of your catalogue — "how many records per type?", "per language?". Callers use facets to build navigation.
Security profile
A bundle of caps applied to every public request: max page size, number of facets, pagination depth. "Prudent" is the default; "Standard" is looser.
Public mode
Who can call /v1/*: anonymous, optional key, or required key.
API key
A random secret assigned to each partner. EGG never stores the raw value — only a hash — and shows it once, on creation.
Rate limit
A cap on how many requests a single caller can make per minute. Keeps the backend safe if an integrator misbehaves.
Draft
The in-progress configuration saved by the setup wizard. It is never applied until you click Publish.
YAML
A plain-text configuration format. The setup wizard avoids asking you to edit it directly.
