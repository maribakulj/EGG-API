A safe, guided public API for your catalogue.

EGG-API wraps a read-only façade around the search backend you already run (Elasticsearch, OpenSearch) and lets a non-technical operator publish it through an eight-step wizard — no YAML, no terminal.

Download the desktop installer Or install from source

Current release: v2.0.0

An eight-step wizard, no YAML

  1. Connect — point EGG at your Elasticsearch/OpenSearch URL and credentials. Click "Test connection".
  2. Source — pick the index or collection to expose. The wizard lists what the backend advertises.
  3. Mapping — the wizard proposes a translation from backend field names to the public shape (id, title, creators…). Adjust with dropdowns.
  4. Security — "Prudent" or "Standard" profile; decide if the public API needs a key.
  5. Exposure — checkboxes for the facets, sorts and fields your API will accept.
  6. Keys — mint a first partner key (shown once). The raw secret is never persisted.
  7. Test — run a live search against the draft config and inspect a sample record.
  8. Publish — swap the running service onto the new config. Done.

Who is this for?

Archivists & librarians

Run an Elasticsearch index somewhere already. Want to publish a safe search API without getting an engineering team involved.

Small institutions

No dedicated DevOps. You want sane defaults (rate limits, CSP, query policy, audit trail) without having to re-implement them.

Partner integrators

Need a stable OpenAPI contract and per-partner keys. EGG never leaks the underlying DSL to the public API.
